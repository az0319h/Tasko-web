# 공지사항 기능 구현 계획

## 접근 권한

### 공지사항 조회

- **공지사항 목록**: 모든 사용자(인증 여부 무관)가 조회 가능
- 공지사항 목록에는 반드시 **작성자 정보**가 표시되어야 함

### 공지사항 작성

- **관리자만** 공지사항 작성 가능
- 일반 사용자는 작성 권한 없음

## 공지사항 생성 기능

### 필수 입력 항목

1. **제목** (필수)
2. **설명/내용** (필수)
3. **이미지 첨부** (1개, 선택)
4. **파일 첨부** (다중, 선택)

### 생성 폼 구성

- 제목 입력 필드
- 설명/내용 입력 필드 (텍스트 에디터)
- 이미지 업로드 필드 (1개 제한)
- 파일 첨부 필드 (다중 파일 업로드 가능)

## 공지사항 표시 방식

### 공지사항 목록

- 목록에는 다음 정보가 표시되어야 함:
  - 제목
  - 작성자 (필수 표시)
  - 작성일시
  - 썸네일 이미지 (있는 경우)

### 공지사항 팝업/상세 보기

팝업으로 표시될 때 다음 순서로 구성:

1. **이미지** (최상단, 있는 경우)
2. **제목**
3. **작성자** (필수 표시)
4. **설명/내용**
5. **첨부 파일 목록** (다운로드 가능한 파일들)

## 구현 고려사항

### 데이터베이스 스키마

- 공지사항 테이블에 작성자 ID 필수
- 이미지 URL 저장 필드
- 첨부 파일 정보 저장 (파일명, URL 등)

### 권한 관리

- RLS 정책 설정:
  - SELECT: 모든 사용자 허용
  - INSERT: 관리자만 허용
  - UPDATE: 관리자만 허용
  - DELETE: 관리자만 허용

### 파일 저장

- 이미지 및 첨부 파일은 Supabase Storage에 저장
- 공개 읽기 권한 설정 (모든 사용자가 다운로드 가능)

### UI/UX

- 팝업은 모달 형태로 구현
- 이미지가 없는 경우 이미지 영역은 표시하지 않음
- 첨부 파일이 없는 경우 파일 목록 섹션은 표시하지 않음
- 작성자 정보는 항상 명확하게 표시
