   - **목적**: 프로젝트 CRUD, 참여자 관리, 프로젝트 목록 조회
   - **핵심 기능**: 프로젝트 생성, 수정, 삭제, 참여자 관리

3. **Task 5**: Task 관리 기능 구현
   - **의존성**: Task 3, Task 4
   - **목적**: Task CRUD, 상태 변경, 칸반 보드 UI
   - **핵심 기능**: Task 생성, 수정, 삭제, 상태 변경, 칸반 보드

4. **Task 6**: 대시보드 및 칸반 보드 UI 구현
   - **의존성**: Task 4, Task 5
   - **목적**: 역할별 대시보드, 칸반 보드 탭 전환, 프로젝트 카드 아코디언

## 🚀 권장 개발 순서 (사용자 제안)

사용자가 제안한 **기본 시나리오**에 따른 개발 순서:

### 1단계: 프로젝트 생성 기능
- **Task 4.3**: 프로젝트 생성 다이얼로그 구현
- **필드**: 기회(title), 고객명(client_name), 완료 예정일(due_date)
- **초대 사용자**: 다중 선택, 필수 (관리자 제외 1명 이상)
- **프로젝트 생성 후**: 목록/상세로 이동

### 2단계: 프로젝트 상세 페이지
- **Task 5.9**: 프로젝트 상세 페이지 구현
- **표시 내용**:
  - 프로젝트 기본 정보 (제목, 고객명, 완료 예정일)
  - 참여자 목록
  - 해당 프로젝트에 속한 Task 목록 (칸반 보드)

### 3단계: Task 생성 기능 (프로젝트 상세 내)
- **Task 5.4**: Task 생성 다이얼로그 구현
- **필드**: 
  - 지시사항(title, 필수)
  - description (선택)
  - task_category (필수, 생성 후 변경 불가)
  - 담당자(assignee, 필수)
  - 마감일(due_date, 필수)
- **생성 즉시**: 프로젝트 상세에 반영

### 4단계: Task 상세 페이지
- **Task 7.5**: Task 상세 페이지 구현
- **표시 내용**:
  - Task 정보 상세 표시
  - Task 상태 전환 (IN_PROGRESS → WAITING_CONFIRM 등)
  - Task 관련 메시지/히스토리 표시

## ⚠️ 주의사항

### Task 2 (인증)는 반드시 먼저 완료해야 함
- 모든 기능이 인증된 사용자만 사용 가능
- 프로필 설정 완료 후에만 다른 기능 접근 가능
- **따라서 실제 개발 순서는**:
  1. **Task 2** (인증) ← 필수 선행
  2. **Task 4** (프로젝트 관리) ← 사용자 제안 순서의 1단계
  3. **Task 5** (Task 관리) ← 사용자 제안 순서의 2~3단계
  4. **Task 7** (채팅) ← 사용자 제안 순서의 4단계
  5. **Task 6** (대시보드) ← UI/UX 개선
  6. **Task 9** (Admin) ← Admin 전용 기능

### Task 3 (DB)는 이미 완료됨
- Phase 1~3 작업으로 완료
- 프론트엔드 개발 시 DB 스키마 변경 불가

### Task 8 (이메일)은 백엔드 작업
- Edge Function 구현
- 프론트엔드에서는 호출만 하면 됨
